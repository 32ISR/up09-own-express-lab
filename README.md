# Проект: Backend для SEO-сайта

## Цель

- Разработать REST API для сайта, который создавали на дисциплине **SEO-оптимизация**.
- Тематика и сущности определяются вами в соответствии с тематикой сайта (блог, магазин, портфолио, новостной сайт и т.д.).
- Пользователи могут регистрироваться, авторизовываться, просматривать публичный контент, а авторизованные — создавать и управлять своими записями.
- Администратор управляет всем контентом и пользователями.
- Внутри репозитория необходимо создать Bruno-коллекцию, описывающую все эндпоинты API.

---

## Перед сдачей создать в БД

- Минимум **5 записей** основной сущности
- Минимум **5 записей** второй сущности
- Администратора с кредами `admin:qwerty123`
- Пользователя с кредами `user:qwerty123`

---

## Выбор тематики

| Тематика сайта | Основная сущность | Вторая сущность |
|---|---|---|
| Блог | Посты (`posts`) | Комментарии (`comments`) |
| Интернет-магазин | Товары (`products`) | Категории (`categories`) |
| Новостной сайт | Статьи (`articles`) | Теги (`tags`) |
| Библиотека / книги | Книги (`books`) | Отзывы (`reviews`) |
| Портфолио | Проекты (`projects`) | Навыки (`skills`) |

> Если ваша тематика не попадает ни в один из этих вариантов — согласуйте сущности со мной.

_Лучше в любом случае уточнить_

## Основные сущности

### 1. `User`

| Поле | Тип | Описание |
|---|---|---|
| `id` | number | Уникальный идентификатор |
| `username` | string | Имя пользователя |
| `email` | string | Email (уникальный) |
| `password` | string | Хэшированный пароль |
| `role` | string | Роль: `"user"` или `"admin"` |
| `createdAt` | date | Дата регистрации |

_Можете добавлять ключи, если нужно_

---

### 2. Основная сущность (например, `Post` / `Product` / `Article`)

Поля определяются исходя из тематики сайта. Обязательные поля:

| Поле | Тип | Описание |
|---|---|---|
| `id` | number | Уникальный идентификатор |
| `title` | string | Название / заголовок |
| `description` | string | Описание / содержимое |
| `createdBy` | number (FK → User.id) | Кто создал запись |
| `createdAt` | date | Дата добавления |

Дополнительные поля добавляются по смыслу (например, `price`, `genre`, `category`, `imageUrl` и т.д.).

---

### 3. Вторая сущность (например, `Comment` / `Review` / `Tag`)

Поля определяются исходя из тематики. Обязательные поля:

| Поле | Тип | Описание |
|---|---|---|
| `id` | number | Уникальный идентификатор |
| `[основная_сущность]Id` | number (FK) | Связь с основной сущностью |
| `userId` | number (FK → User.id) | Автор записи |
| `createdAt` | date | Дата создания |

Например, если вторая сущность — комментарий, добавьте поле `text`; если это отзыв — `rating` (1–5) и `comment`.

---

## Аутентификация

Использовать **JWT (JSON Web Token)**.  
Токен выдаётся при логине и передаётся в заголовке каждого защищённого запроса:

```
Authorization: Bearer <token>
```

## Структура проекта

```
my-project/
├── db/
│   ├── database.js      # инициализация БД
│   └── seed.js          # начальные данные (5 записей, пользователи)
├── middleware/
│   ├── auth.js          # проверка JWT
│   └── checkRole.js     # проверка роли
├── routes/
│   ├── auth.js
│   ├── [сущность].js
│   └── admin.js
├── bruno/               # Bruno-коллекция с эндпоинтами
├── .env
├── .gitignore
├── app.js
└── package.json
```

> В `.gitignore` обязательно добавьте `node_modules/`, `.env` и файл базы данных (`*.db`), но при этом файл БД должен быть создан при запуске `seed.js`.

## Как сдавать

- Создайте форк репозитория в вашей организации с названием `[название-репозитория]-[ваша-фамилия]`
- Используя ветку `wip`, выполните задание
- Зафиксируйте изменения в вашем репозитории
- Когда работа будет готова — создайте Pull Request из ветки `wip` в ветку `main` (обе в вашем форке) и укажите меня (`ktkv419`) в качестве reviewer

**Не мержите PR самостоятельно** — это сделает преподаватель после проверки.
